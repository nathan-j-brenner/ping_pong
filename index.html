Ping Pong

  1. Find a partner. Determine role: Ping or Pong
  2. Ping: create a new repo
    1. Initialize with readme.md
    2. Click on settings, then collaborator. Add your partner’s gitHub username.
    3. Git clone url
  3. Pong:
    1. In terminal: git clone url
    2. Git branch: You should be on master
    3. Git branch [your name]
    4. Git checkout [your name]
    5. Open the repository in sublime
    6. Create a new file
    7. Save file as index.html
    8. In file, type html then hit tab
    9. Save
    10. Git add index.html
    11. Git commit -m "ping 1"
    12. Git checkout [master]
    13. Git merge [your name]
    14. In gitHub, create new branch with your name
    15. Git push
    16. Confirm your push with your partner
  4. Ping: git clone url
    1. Wait for your partner to push
    2. Git pull
    3. Git branch [your name]
    4. Git checkout [your name]
    5. Create a paragraph tag that says "[Your name] is ping, and ping is awesome"
    6. From GitHub, create a new branch that is your name.
    7. Save, git add, git commit, git checkout master, git merge [your name], git push
    8. Tell your partner that you’ve pushed
  5. Pong: create a conflict and resolve
    1. Git pull
    2. Make sure you’re on your branch
    3. In the paragraph that says "[partern’s name] is ping, and ping is awesome", change to "[partner’s name] is ping, [your name] is pong, and ping and pong is awsome".
    4. Save, git add/commit
    5. Resolve the conflict
  6. Ping: create another conflict and resolve
